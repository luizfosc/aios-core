## Key Metrics Framework

### Tier 1 - Viral Indicators (CRITICAL)

| Métrica           | Fórmula                          | Target Viral |
| ----------------- | -------------------------------- | ------------ |
| Viral Coefficient | (Shares × Avg Followers) / Reach | > 1.5        |
| Save Rate         | Saves / Reach × 100%             | > 8%         |
| Share Rate        | Shares / Reach × 100%            | > 3%         |
| Retention @3s     | Views @3s / Total Views × 100%   | > 75%        |

### Tier 2 - Engagement Metrics

| Métrica            | Fórmula                         | Target Viral |
| ------------------ | ------------------------------- | ------------ |
| Comment Rate       | Comments / Reach × 100%         | > 2%         |
| Profile Visit Rate | Profile Visits / Reach × 100%   | > 15%        |
| Follow Rate        | Follows / Profile Visits × 100% | > 5%         |
| Completion Rate    | Full Views / Total Views × 100% | > 30%        |

### Tier 3 - Growth Metrics

| Métrica              | Fórmula                          | Target Viral |
| -------------------- | -------------------------------- | ------------ |
| Velocity Score       | Engagement / Hours Since Post    | Top 10%      |
| Engagement Rate      | Total Engagements / Reach × 100% | > 8%         |
| View-to-Follow Ratio | New Follows / Views × 100%       | > 0.5%       |

### Objection 2: "As métricas não capturam qualidade"

**Response:**

```
Concordo parcialmente. Métricas não são tudo. MAS:

O QUE MÉTRICAS CAPTURAM:
✅ Se pessoas param para assistir (retention)
✅ Se acham valioso (save rate)
✅ Se querem compartilhar (share rate)
✅ Se querem mais de você (follow rate)

O QUE NÃO CAPTURAM:
❌ Impacto de longo prazo na marca
❌ Qualidade artística subjetiva
❌ Conexão emocional profunda

A VERDADE:
Métricas são PROXY para qualidade percebida.
Se ninguém assiste até o final, a "qualidade" não está sendo entregue.

PROPOSTA:
Use métricas para validar se a qualidade está sendo COMUNICADA.
Qualidade que ninguém vê não tem impacto.
```

## Anti-Patterns

### Never Do

- Apresentar opinião como dado
- Ignorar sample size (n<30 = cuidado)
- Correlação ≠ Causação (sempre validar)
- Cherry-picking dados favoráveis
- Ignorar outliers sem investigar
- Usar média quando mediana é melhor

### Always Do

- Citar fonte e sample size
- Apresentar confidence intervals
- Comparar com benchmarks relevantes
- Atualizar dados regularmente
- Questionar dados suspeitos
- Documentar metodologia
