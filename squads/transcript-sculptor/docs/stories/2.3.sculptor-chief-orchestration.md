### Merge Algorithm — Overlap Resolution

```
Chunk N ends:   "...e é por isso que este conceito é tão importante para vocês."
Chunk N+1 starts: "é tão importante para vocês. Agora vamos para o próximo tópico..."
→ Detect overlap via string similarity
→ Keep Chunk N version, start Chunk N+1 after overlap
```

### Key Design Decisions

1. **Overlap Removal Strategy:**
   - Fuzzy matching first 100 words of chunk N+1 with last 300 words of chunk N
   - Remove overlap from N+1 (keep N's version for continuity)
   - Threshold: 80% word sequence match = overlap detected

2. **TOC Anchor Format:**
   - GitHub-compatible: lowercase, spaces → hyphens, remove special chars
   - Handles brackets: "[Q&A] Perguntas" → "#qa-perguntas"

3. **Metadata Inference Hierarchy:**
   - Title: analysis-map zone title → first H2 → source filename
   - Duration: word_count / 150 (PT-BR speaking rate)
   - Speakers: from analysis-map labels

4. **Conflict Resolution Priorities:**
   - Duplicate headings: keep first, merge content
   - Split dialogues: merge under single speaker attribution
   - Abrupt transitions: preserve if natural, mark `[TRANSIÇÃO]` if jarring
