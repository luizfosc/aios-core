## Dev Notes

### Workflow Architecture (from architecture doc)

The workflow follows the **Pipeline Pattern** with 5 sequential stages:

```
Input Folder → Stage 0 (Ingestion) → Stage 1 (Analysis) → Stage 2 (Editing) → Stage 3 (QA) → Stage 4 (Delivery) → masterpiece.md
```

Each stage produces intermediate artifacts in the output folder structure:

```
{output_folder}/
├── raw/                           # Stage 0 output
├── analysis/                      # Stage 1 output
├── chunks/                        # Stage 2 output
├── reports/                       # Stage 3 output
├── ingestion-report.yaml          # Stage 0 summary
├── processing-log.yaml            # Pipeline timing/status
├── pipeline-state.yaml            # Resumable state
└── {source}-masterpiece.md        # FINAL OUTPUT
```

### Partial Reprocessing

Idempotent stages enable partial reprocessing:

| Entry Point | Skips | Uses Existing |
|-------------|-------|---------------|
| `--from=stage0` | Nothing | Full run |
| `--from=stage1` | Ingestion | `raw/*.md` from previous run |
| `--from=stage2` | Ingestion + Analysis | `raw/*.md` + `analysis-map.yaml` |
| `--from=stage3` | Ingestion + Analysis + Editing | All chunks + analysis |

### Architecture Decisions

1. **Idempotent Stages:** Each stage can be re-executed independently using cached inputs from previous stages
2. **Chunk-and-Merge Pattern:** Long content split into processable chunks (5k-8k words), edited independently, merged intelligently
3. **HITL Escape Hatch:** Ambiguous content marked `[REVISAR]` instead of arbitrary decisions
4. **Defense in Depth:** Quality gates at multiple levels (ingestion checklist, editorial checklist, QA gate, final delivery gate)
5. **Graceful Degradation:** Non-critical errors warn and continue rather than blocking entire pipeline
