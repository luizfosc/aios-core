## Dev Notes

### Smoke Test Expectations

**Short sample (~1k words):**
- Single chunk (no splitting needed)
- Expected pipeline time: < 3 minutes
- Expected content preservation: >= 98%
- Expected scores: all >= 7.0

**Long sample (~10k words):**
- 2-3 chunks expected (at ~5000-8000 words each)
- Expected pipeline time: < 15 minutes (NFR1)
- Expected content preservation: >= 98%
- Merge transitions should be seamless (no repeated content, no gaps)

### Test Sample Requirements

Samples must include:
- PT-BR language
- At least 2 identifiable speakers
- At least 2 different zones (lecture + Q&A minimum)
- Some noise candidates (murmurs, [inaudivel], loose phrases)
- Natural topic changes for title suggestion testing

### Slash Command Pattern

Follow existing squad patterns (e.g., `content-engine`, `knowledge-base-builder`):

```
.claude/commands/transcript-sculptor/
├── README.md           # Squad overview + command list
├── sculptor-chief.md   # → squads/transcript-sculptor/agents/sculptor-chief.md
├── doc-converter.md    # → squads/transcript-sculptor/agents/doc-converter.md
├── transcript-analyst.md
├── transcript-editor.md
├── quality-guardian.md
└── process.md          # → squads/transcript-sculptor/workflows/wf-transcript-to-masterpiece.md
```
