### Analysis Map Schema (from architecture doc)

```yaml
analysis_map:
  source_file: "raw/filename.md"
  total_words: 42350
  language: "pt-BR"
  speakers:
    - id: "speaker-1"
      label: "Palestrante Principal"
      estimated_percentage: 85
      tone: "energetic"
      characteristics: ["usa 'galera', 'olha só'"]
  zones:
    - type: "lecture|qa|music|pitch|noise|break"
      start_word: 0
      end_word: 12000
      title_suggestion: "Section Title"
  chunks:
    - id: "chunk-001"
      start_word: 0
      end_word: 7500
      zones: ["lecture"]
      overlap_context: "last ~200 words..."
  noise_candidates:
    - start_word: N
      end_word: N
      text_preview: "..."
      confidence: 0.92
      recommendation: "remove|review"
```

### Speaker Detection Heuristics

- Dialogue markers: "—", ":", quotes preceding speech
- Vocabulary shifts: sudden change in register/formality
- Explicit markers: "Pergunta:", "Resposta:", "Participante:"
- Tone shifts: from teaching to answering mode

### Zone Detection Patterns

| Zone | Indicators |
|------|-----------|
| Lecture | Continuous monologue, teaching tone |
| Q&A | Question-answer pattern, multiple speakers |
| Music | Lyrics, song references, "[música]" markers |
| Pitch | Sales language, product mentions, CTAs |
| Noise | "[inaudível]", very short isolated phrases, murmurs |
| Break | "intervalo", "voltamos", "pausa" |

### Key Design Decisions

1. **LLM-Executable Task Definition:** The task file is designed to be followed step-by-step by Claude Code (or any LLM), not a Python script. Each step has clear objectives, detection patterns, and examples.

2. **Word-Level Boundaries:** Chunking uses word positions (0-indexed) rather than character offsets for cleaner splitting and better alignment with natural language boundaries.

3. **Overlap Strategy:** 200-word overlap between chunks ensures context continuity during editing and final merge, preventing semantic breaks at chunk boundaries.

4. **Confidence-Based Noise Detection:** Two-tier system (remove vs review) prevents over-aggressive removal while flagging ambiguous content for HITL review.

5. **Zone Detection Patterns Table:** Comprehensive table in Step 3 makes zone classification deterministic and repeatable.
