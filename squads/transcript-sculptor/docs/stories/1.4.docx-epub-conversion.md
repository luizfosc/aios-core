## Story

**As a** user,
**I want** Word documents, EPUB, and other formats converted to markdown preserving structure,
**so that** all materials are in uniform format for the intelligence pipeline.

## Acceptance Criteria

1. DOCX → MD via Pandoc preserving headings, lists, and tables
2. EPUB → MD via Pandoc preserving chapters
3. RTF and ODT → MD via Pandoc
4. SRT/VTT → MD with timestamps converted to context markers
5. Embedded images extracted to `{output}/attachments/` folder

## Dev Notes

### Pandoc Commands

```bash
# DOCX → MD with images
pandoc document.docx -t markdown_strict --extract-media='./attachments' -o document.md

# EPUB → MD
pandoc ebook.epub -t markdown_strict -o ebook.md

# RTF → MD
pandoc document.rtf -t markdown_strict -o document.md

# ODT → MD
pandoc document.odt -t markdown_strict -o document.md
```

### SRT Format Example

```
1
00:00:01,000 --> 00:00:04,000
Bem-vindos ao nosso curso.

2
00:00:04,500 --> 00:00:08,000
Hoje vamos falar sobre o método.
```

Should become:

```markdown
[00:00:01] Bem-vindos ao nosso curso.

[00:00:04] Hoje vamos falar sobre o método.
```

### Attachments Folder

Images extracted go to `{output}/attachments/{source-filename}/` to avoid collisions between files.

### Technical Decisions

1. **Pandoc format:** Used `markdown` instead of `markdown_strict` to preserve more structure (tables, etc.)
2. **Image extraction:** Organized by source filename to prevent collisions: `attachments/{source-filename}/`
3. **Subtitle grouping:** 2-second gap threshold balances readability vs. preserving natural pauses
4. **Timestamp format:** [HH:MM:SS] provides context without cluttering every line
5. **HTML tag removal:** Regex-based removal handles common subtitle formatting tags
6. **Error handling:** All functions return status dicts, allowing batch processing to continue on errors

### Edge Cases Handled

- Pandoc not installed → Error returned with clear message
- Corrupt files → Subprocess errors caught and returned as error status
- Empty subtitle files → Handled gracefully, returns 0 blocks
- Password-protected DOCX → Pandoc will fail, error captured and returned
- VTT optional hours → Regex handles both HH:MM:SS.mmm and MM:SS.mmm formats
- SRT trailing info on timestamps → Split and take first part only
