# Ensinio — Playbook de Retenção (Comunidade + Feed + Gamificação)

> Como usar comunidade, feed e gamificação como sistema integrado de retenção.
> Para: CS orientar cliente, closer usar como argumento, ou cliente implementar sozinho.
> Fonte: Vídeo "Plataforma e App para Infoprodutores" — Gui Ávila (2026-03-14)

---

## Por Que Retenção Importa Mais Que Venda

Gui Ávila resume assim:
> "Aula é commodity. Experiência é premium."

O infoprodutor médio foca 90% em aquisição e 10% em retenção. A Ensinio inverte essa lógica: com gamificação, comunidade e feed, o aluno **não quer sair** — e isso gera:
- Menos churn (cancelamento)
- Mais LTV (lifetime value)
- Mais recompra (upsell para base ativa)
- Mais indicações orgânicas

### Efeito Posse (Gui Ávila — livro "Shortcuts")
Quando o aluno acumula pontos, selos, moedas e sobe de nível, ele sente **posse** daquilo. Psicologia social: ninguém quer abrir mão do que conquistou. Resultado: engajamento aumenta, churn diminui.

## O Sistema de Retenção Ensinio (3 Pilares)

```
┌─────────────────────────────────────────────────┐
│              BILLBOARD (vitrine)                 │
│  Aluno entra → vê conteúdo → quer participar    │
└──────────────────────┬──────────────────────────┘
                       │
        ┌──────────────┼──────────────────┐
        ▼              ▼                  ▼
  ┌──────────┐  ┌────────────┐  ┌──────────────┐
  │COMUNIDADE│  │   FEED     │  │ GAMIFICAÇÃO  │
  │ (grupo)  │  │ (central)  │  │ (plataforma) │
  │          │  │            │  │              │
  │ Posts    │  │ Centraliza │  │ Pontos       │
  │ Dúvidas  │  │ atividades │  │ Níveis       │
  │ Reações  │  │ de todos   │  │ Ranking      │
  │          │  │ os grupos  │  │ Recompensas  │
  └──────────┘  └────────────┘  └──────────────┘
        │              │                  │
        └──────────────┼──────────────────┘
                       ▼
              ALUNO ENGAJADO
         (fica mais, compra mais)
```

## Pilar 1: Comunidade no Grupo

### Configuração básica
- Cada grupo já vem com comunidade ativada (desativar se não quiser)
- Página inicial do grupo: escolher **Comunidade** como destino (incentiva participação antes de assistir aula)
- Capa da comunidade: banner visual no topo (identidade do grupo)

### Estratégias de uso

**Para cursos:**
- Página inicial = Comunidade → aluno vê posts antes de assistir aula
- Comentários na aula aparecem também no feed → gera sensação de "vida" na plataforma
- Comentários privados (aluno + professor) para dúvidas sensíveis
- Comentários públicos (todos os membros do grupo) para engajamento social

**Para mentorias:**
- Grupo com comunidade apenas (sem aulas)
- Mentorado posta atualizações, mentor responde
- Cria senso de grupo e accountability

**Para assinaturas:**
- Comunidade como diferencial do plano premium
- "Quem assina tem acesso à comunidade exclusiva"
- Reter assinante mesmo quando não assiste aula

### Moderação
- Central de comentários: Admin Console → Grupos e Recursos → Comentários
- Revisados vs não revisados
- Excluir, responder, ver aula de origem
- Configurável: permitir comentário público, privado ou ambos

## Pilar 2: Feed Central

### O que é
O feed (/feed) é a **rede social** da plataforma. Centraliza atividades de todas as comunidades em um lugar só. O aluno não precisa entrar em cada grupo para ver o que está acontecendo.

### Configuração
- Admin Console → Configurações → Comunidade
- Ativar feed: **sim** (recomendado)
- Permissão: **Todos os usuários** ou **Membros de grupos específicos**
- Criar sessões no feed: produtos em destaque, conteúdo gratuito, etc.

### Segmentação inteligente
O feed é "público" no acesso, mas **segmentado** no conteúdo:
- Post feito na comunidade do Curso A → só aparece para quem tem acesso ao Curso A
- Post feito direto no feed (para todos) → aparece para todos
- Isso significa: feed aberto não vaza conteúdo privado

### Estratégias de uso

**Feed como vitrine de engajamento:**
- Aluno entra no feed → vê atividade dos grupos que participa
- Vê que outros alunos estão postando → FOMO → volta a participar

**Feed como canal de vendas suave:**
- Criar sessão no feed com produtos à venda
- Aluno que entrou para ver comunidade vê os produtos → cross-sell natural

**Posts com destaque (só no feed):**
- Posts no feed suportam título + imagem banner (diferente da comunidade do grupo)
- Usar para avisos importantes, lançamentos, eventos

### Perfil do aluno no feed
- Foto, capa, resumo de curtidas e publicações
- Nível de gamificação visível
- Aulas em andamento ("continuar assistindo")
- Gera senso de identidade e pertencimento

## Pilar 3: Gamificação como Motor de Retenção

### Configuração recomendada (nível plataforma)

**Ativar:** Admin Console → Grupos e Recursos → Gamificação → Ativar

**Gatilhos recomendados para começar:**

| Gatilho | Pontos | Moedas | Limite diário | Por quê |
|---------|--------|--------|---------------|---------|
| Login diário | 1 | 0 | 1 | Hábito |
| Assistir aula | 2 | 1 | 3 | Conclusão |
| Comentar em aula | 1 | 0 | 2 | Engajamento |
| Comentar na comunidade | 1 | 0 | 2 | Interação social |
| Reagir a publicação | 1 | 0 | 3 | Baixo atrito |
| Acertar quiz | 3 | 2 | 5 | Aprendizado |
| Enviar tarefa | 5 | 3 | 1 | Comprometimento |

**Limites diários são essenciais** — evitar que alguém abuse (10 logins = 10 pontos).

**Níveis sugeridos:**

| Nível | Pontos | Adaptação por nicho |
|-------|--------|-------------------|
| Iniciante | 2 | "Curioso" (educação), "Aprendiz" (mágica) |
| Bronze | 50 | "Praticante" |
| Prata | 100 | "Dedicado" |
| Ouro | 200 | "Expert" |
| Esmeralda | 400 | "Mestre" |
| Safira | 600 | "Sensei" |
| Rubi | 800 | "Lenda" |
| Diamante | 1000 | "Imortal" |

**Dica do Gui:** Personalize os nomes dos níveis por nicho. Plataforma de finanças? "Nível Warren Buffett". Mágica? "Nível Houdini". Isso gera identificação.

### Ranking
- Ativar ranking geral (aparece na Billboard como opção)
- Competição saudável entre alunos
- Top alunos se sentem valorizados → promotores da marca

### Loja de recompensas (quando estiver pronto)
- Ativar quando tiver prêmios definidos
- Pode começar simples: cupom de desconto, acesso a conteúdo bônus
- Evoluir para: produtos físicos (camiseta, livro), experiências (call com o expert)
- Cada recompensa: moedas necessárias, estoque limitado, data de expiração, limite por usuário
- Restringir por grupo (ex: só membros do plano premium podem resgatar)

**Dica:** Se não tem prêmios ainda, ative pontos e níveis, mas desative a loja. Alunos acumulam pontos → quando abrir a loja, já têm saldo → rush de resgate.

## Combinações Poderosas

### Combo 1: Comunidade + Gamificação = Engajamento Orgânico
- Aluno posta na comunidade → ganha pontos
- Outros alunos reagem → ganham pontos
- Todos sobem de nível → notificação com confetes
- **Resultado:** Loop de engajamento auto-sustentável

### Combo 2: Feed + Vendas = Cross-Sell Natural
- Aluno entra no feed para interagir
- Vê sessão de produtos na lateral
- Vê que outros alunos estão no Curso B
- **Resultado:** Compra sem pressão de vendas

### Combo 3: Gamificação + Drip Content = Anti-Reembolso
- Liberar aulas a conta-gotas (drip content)
- Gamificar conclusão de aulas (pontos por assistir)
- Liberar bônus apenas após período de garantia
- **Resultado:** Aluno engajado antes do prazo de reembolso → não cancela

### Combo 4: Comunidade + Agente Tutor = Suporte Zero Custo
- Aluno posta dúvida na comunidade → agente tutor responde
- Monitores humanos só para casos complexos
- **Resultado:** Suporte 24/7 sem headcount

### Combo 5: Ranking + Cupom Inteligente = Upsell por Mérito
- Top 10 alunos no ranking recebem cupom exclusivo de upgrade
- Cupom configurado: "só para quem tem acesso ao Curso A"
- **Resultado:** Premiar os mais engajados → eles compram mais E se sentem valorizados

## Métricas de Retenção (Admin Console → Dados)

| Métrica | Onde ver | O que indica |
|---------|----------|-------------|
| Taxa de conclusão | Dados → [grupo] | Alunos estão terminando o curso? |
| CSAT por aula | Dados → Satisfação | Quais aulas precisam melhorar? |
| Engajamento médio | Dados → Estatísticas | Alunos estão voltando? |
| Horário de acesso | Dados → Estatísticas | Quando publicar no feed/comunidade |
| Ranking de gamificação | Dados → Gamificação | Quem são os promotores da marca? |
| Usuários mais engajados | Dados → Desempenho | Candidatos para depoimento/case |

## Checklist: Retenção Mínima Viável

- [ ] Comunidade ativa em pelo menos 1 grupo
- [ ] Página inicial do grupo = Comunidade (para incentivar participação)
- [ ] Feed ativo com pelo menos 1 sessão de produtos
- [ ] Gamificação ativa com 3+ gatilhos e 5+ níveis
- [ ] Ranking geral ativado
- [ ] Pelo menos 1 post do produtor no feed (dar o exemplo)
- [ ] Testar: entrar como aluno e interagir → pontos sobem → nível muda
