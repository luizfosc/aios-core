# Task: Create Offer Stack

**Task ID:** create-offer
**Executor:** Agent (dan-kennedy)
**Execution Type:** Agent
**Quality Gate:** QG-003
**Template:** templates/offer-stack-tmpl.md

---

## Purpose

Criar uma oferta irresistivel usando os principios de Dan Kennedy: core offer + bonus stack + garantia + urgencia + escassez.

---

## Inputs

| Input           | Required | Description                   |
| --------------- | -------- | ----------------------------- |
| product         | Yes      | Produto/servico principal     |
| market          | Yes      | Publico-alvo                  |
| price           | Yes      | Preco desejado                |
| competitors     | No       | Ofertas concorrentes          |
| existing_assets | No       | Bonus/materiais ja existentes |

---

## Steps

1. **Define Core Offer** - O que o cliente recebe (resultado, nao produto)
2. **USP Development** - Responder "Por que voce vs todos os outros?"
3. **Build Bonus Stack** - 3-5 bonus que amplificam o valor
4. **Price Anchoring** - Comparar com alternativas mais caras
5. **Value Calculation** - Somar valor total vs preco
6. **Guarantee Design** - Risk reversal (quanto mais forte, melhor)
7. **Urgency Element** - Deadline real (nao artificial)
8. **Scarcity Element** - Limitacao real (vagas, estoque, tempo)
9. **Payment Options** - Reduzir friccao (parcelamento, trial)
10. **No BS Audit** - 10 Rules de Kennedy aplicadas
11. **Handoff** - Para copywriter executar ou Torriani validar

---

## Acceptance Criteria

- [ ] Core offer claro (resultado, nao features)
- [ ] USP responde "por que voce?"
- [ ] Bonus stack com valor > 3x preco
- [ ] Garantia presente e forte
- [ ] Urgencia real (nao fake countdown)
- [ ] Escassez justificada
- [ ] Preco ancorado (comparacao)
- [ ] Instrucoes claras de como comprar
- [ ] Trackavel (como medir resultado)

## Veto Conditions

- **BLOCKER:** Core offer focado em features em vez de resultado/transformação
- **BLOCKER:** USP ausente ou genérica — não responde "por que VOCÊ e não o concorrente?"
- **BLOCKER:** Ausência de garantia — oferta sem garantia transmite insegurança
- **WARNING:** Bonus stack com valor percebido menor que 3x o preço
- **WARNING:** Escassez fabricada (countdown fake, vagas inventadas) em vez de escassez real
