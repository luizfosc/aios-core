# Task: Diagnose High-Ticket

## Metadata
- **Type:** Diagnostic
- **Specialist:** Leandro Ladeira
- **Duration:** 15-20min
- **Activation:** `/leandro-ladeira:leandro-ladeira *diagnose-campanha`

## Purpose

Diagnosticar o negócio/campanha usando o framework M.O.E.R (Mensagem-Oferta-Engate-Retórica) para identificar gaps no pipeline high-ticket.

## Inputs

- Descrição do negócio/produto
- Público-alvo atual
- Campanha atual (se existir)
- Resultados atuais (métricas)

## Process

1. Ativar Leandro Ladeira: `/leandro-ladeira:leandro-ladeira`
2. Executar comando: `*diagnose-campanha`
3. Responder perguntas do diagnóstico M.O.E.R
4. Receber análise de gaps

## Outputs

- Diagnóstico M.O.E.R completo:
  - **M**ensagem: Está clara? Conecta com dor real?
  - **O**ferta: Está irresistível? Diferenciada?
  - **E**ngate: Captura atenção nos primeiros 3 segundos?
  - **R**etórica: Linguagem persuasiva? Storytelling presente?
- Lista de gaps identificados
- Recomendação de fases prioritárias do pipeline

## Checkpoint

**Condition:** Diagnóstico completo com gaps identificados

**Veto Conditions:**
- Diagnóstico genérico (não específico ao negócio)
- Gaps não priorizados

## Handoff

- **If gaps críticos:** Recomendar pipeline completo (todas as 8 fases)
- **If gaps pontuais:** Recomendar fases específicas (ex: só posicionamento)
- **If ready:** Validar que está pronto para tráfego (fase 6)
