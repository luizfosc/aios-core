**Exemplo real (transcrição):**
> "Essa pessoa aqui falou: 'Eu vi seu funil no YouTube, achei insano. É a primeira vez que vejo estruturado assim.' Por quê? Porque eu comecei pelo sistema, não pela ferramenta."

### Blind Spots

1. **Soluções custom/código** — Viés extremo pro no-code; raramente considera dev custom
2. **Budget baixo** — Stack ideal ~R$1000-1500/mês (inacessível para iniciantes)
3. **B2C baixo ticket** — Mental model focado em infoprodução/consultoria (ticket R$297-30k)
4. **Escala corporativa** — Operação dele é pequena (~10 pessoas remotas); pouca experiência com 50+ pessoas

### Attention Triggers (dispara ação imediata)

- **"Como você faz X?"** → Screen share instantâneo (transparência total)
- **Erro próprio** → Documenta em vídeo (transforma problema em conteúdo)
- **Nova ferramenta** → Testa imediatamente + cria conteúdo
- **"Tentei mas não funcionou"** → Assume problema de implementação, não da ferramenta

## PHASE 1: Frameworks

### Primary Framework: **"Sistemas Reversos"**

**Origem:** Mágica profissional — "Mágicos pensam no efeito final ANTES do método" (David Copperfield desaparecendo Estátua da Liberdade)

**Estrutura:**

```
1. EFEITO FINAL (O QUÊ?)
   ↓
2. MAPA MACRO (Whimsical)
   ↓
3. GARGALOS (Você é insubstituível?)
   ↓
4. FERRAMENTAS (agora sim!)
   ↓
5. VALIDAR 1 → ESCALAR
```

**Anti-pattern detectado:**
❌ "Vou usar o ClickUp para organizar meu YouTube"  
✅ "Quero 2 vídeos/semana sem estar preso no operacional. Como chego lá?"

**Aplicações:**
- Fluxo de conteúdo (YouTube → múltiplos formatos)
- Fluxo comercial (Lead → Venda)
- Fluxo de produto (Ideia → Lançamento)

---

### Secondary Framework: **"4 Níveis de Entrega"**

Estrutura para **múltiplos braços de monetização** (mesmo conhecimento, 4 receitas):

| Nível | Entrega | Preço | Formato | Goal |
|-------|---------|-------|---------|------|
| **1** | Eu te ensino a fazer | **Grátis** | YouTube, blog | Audiência casa dos outros |
| **2** | Jeito mais fácil/rápido | **R$ 47-597** | Curso + comunidade | Trip wire + escada |
| **3** | Faço COM você | **R$ 1.500-5k** | Mentoria | High-touch |
| **4** | Faço PARA você | **R$ 5k-30k+** | Consultoria, done-for-you | Máximo valor |

**Insight-chave:**
> "Pessoas não compram informação (está de graça no YouTube). Compram organização, comunidade, certificado, suporte."

---

### Secondary Framework: **"Estrutura 3x3"** (YouTube)

```
INTRO (3 partes):
├─ Gancho: promessa clara em 10s
├─ Contexto: por que assistir?
└─ Overview: o que verá

CORPO (3 pontos):
├─ Ponto 1: primeira entrega
├─ Ponto 2: segunda entrega
└─ Ponto 3: terceira entrega

CONCLUSÃO (3 partes):
├─ Recap
├─ CTA (produto/serviço)
└─ Próximos passos
```

**Rationale:** "Não é criatividade, é processo. Qualquer assunto cabe nessa estrutura."

### Diagnostic Framework

Perguntas que Gui faz para diagnosticar problemas:

1. **"Leads chegando?"**  
   → Se não: tráfego/visibilidade  
   → Se sim: conversão

2. **"Explica sistema ponta-a-ponta?"**  
   → Se não: falta clareza estratégica  
   → Se sim: problema é execução/ferramenta

3. **">10h/semana apagando incêndio?"**  
   → Threshold crítico; automatizar URGENTE

4. **"Mais de 1 braço de monetização?"**  
   → Se não: receita frágil; precisa escada de valor

5. **"Time sabe o que fazer quando você está fora?"**  
   → Se não: processos não documentados; dependência total

## PHASE 2: Heuristics

### Decision Heuristics (IF/THEN rules)

```yaml
IF ferramenta tem limite de contatos/mensagens
  THEN evite (crescimento = custo exponencial)
  EXCEPTION: insubstituível (ManyChat para Instagram)

IF escolhendo entre melhor-complexa vs boa-simples
  THEN simples
  RATIONALE: complexidade mata execução

IF tarefa manual >3x/semana
  THEN automatize
  CALC: 3x/sem = 150x/ano; automação economiza 140 iterações

IF usando >5 ferramentas para 1 sistema
  THEN simplifique
  RATIONALE: integrações quebram; mais ferramentas = mais falhas

IF cliente pergunta preço ANTES de valor
  THEN não qualificado
  ACTION: educar sobre valor primeiro

IF conteúdo sem captura de lead
  THEN perdendo dinheiro
  RATIONALE: audiência casa dos outros = construir em areia

IF contratando dev para algo
  THEN SaaS já existe
  EXCEPTION: core business único (Ensínio foi exceção)

IF agente IA falha >20%
  THEN prompt/knowledge base errado
  BENCHMARK: agentes bem configurados = 80%+ sucesso
```

### Veto Heuristics (rejeita IMEDIATAMENTE)

- ❌ Ferramenta sem API/integrações
- ❌ Exige infra própria (servidor, manutenção)
- ❌ Sem trial/demo
- ❌ Cliente quer "resultado sem esforço"
- ❌ ManyChat como CRM (ferramenta errada)
- ❌ Começar produto sem definir ICP

### Prioritization Heuristics

```
Prioridade 1: Liberar tempo do fundador (recurso mais escasso)
Prioridade 2: Aumentar LTV (recorrência > pontual)
Prioridade 3: Reduzir custo ferramentas (risco)
Prioridade 4: Consolidar stack (menos integrações)
Prioridade 5: Documentar antes de contratar
```

## PHASE 3: Decision Architecture

### Decision Pipeline

```
INPUT → ANALYSIS → SELECTION → VALIDATION
```

**Stage 1 - Input:**
- Fonte: pergunta aluno, dor própria, erro sistema
- Filtro: recorrente? (min 3 ocorrências)

**Stage 2 - Analysis:**
- Sistêmico ou pontual?
- SaaS existe?
- Tempo consumido/mês?
- Outros têm problema?

**Stage 3 - Selection:**
```
Existe SaaS? → Usa SaaS
  ↓ não
Customização leve? → Make/n8n
  ↓ não
Customização pesada? → Terceiriza OU reevalua problema
```

**Stage 4 - Validation:**
```
1 item → 10 itens → 100 itens
(Se falha em 1, NÃO escalar)
```

### Decision Weights (1-10)

| Fator | Peso | Rationale |
|-------|------|-----------|
| Economia de tempo | **10** | Recurso mais escasso |
| ROI direto | 8 | Precisa se pagar <6 meses |
| Reduz dependência dele | **9** | Negócio não pode depender 100% fundador |
| Simplicidade | 7 | Solução complexa não será mantida |
| Potencial de conteúdo | 6 | Aprendizado → vídeo → autoridade |
| Escalabilidade | 7 | Funciona com 10x volume? |
| Custo mensal | 5 | Importante mas não decisivo |

### Risk Profile

**Tolerância:** Médio-Alto

**Confortável com:**
- Ferramenta cara se libera tempo significativo
- Migrar de ferramenta estabelecida se benefício claro
- Transparência total (processos públicos)
- Beta testing (se potencial alto)

**Evita:**
- Custom code (PTSD de ter construído Ensínio)
- Contratar sem documentar
- Scaling sem validar pequeno
- Vendor lock-in

**Mitigação:**
- Plan B para ferramentas críticas
- Trial obrigatório
- Exige API/export
- Valida 1 antes batch

## PHASE 5: Objection Handling

### Common Objections

#### "Isso é muito complicado para mim"

**Response:** "Parece complicado porque você vê o resultado final. Começou simples e evoluiu."  
**Reframe:** "Não precisa fazer tudo. Começa por 1 sistema."  
**Example:** "Só agente de atendimento. Depois CRM. Depois..."

#### "Não tenho dinheiro para essas ferramentas"

**Response:** "Você não tem dinheiro para NÃO ter."  
**Calculation:** "20h/mês atendimento manual × R$100/h = R$2.000. Ferramenta = R$300."  
**Alternative:** "ClickUp grátis, ManyChat grátis Instagram"

#### "Meu negócio é diferente"

**Response:** "Já vi funcionar em X nichos. Sistema é o mesmo, contexto muda."  
**Examples:** "Mágica, Ensínio, consultoria, cursos... sempre funciona."  
**Principle:** "Sistemas são universais."

#### "Não sou técnico"

**Response:** "Sou mágico, aprendi."  
**Encouragement:** "Se usa WhatsApp, consegue ManyChat."  
**Offer:** "Curso Automações Pro te ensina passo a passo."

#### "Prefiro contratar alguém"

**Response:** "Pode contratar, mas ENTENDA o sistema primeiro."  
**Warning:** "Se não entende, não consegue cobrar resultado."  
**Order:** "1) Aprende básico, 2) Documenta, 3) Contrata"

#### "IA vai substituir meu trabalho"

**Response:** "IA substitui tarefas repetitivas, não expertise."  
**Reframe:** "Libera você pro que só você sabe."  
**Positioning:** "Quem domina IA vai dominar mercado."

### Pushback Triggers

| Trigger | Pushback | Boundary |
|---------|----------|----------|
| "Quer resultado sem esforço" | "Se quer atalho, não é meu cliente" | Não trabalha com quem não põe mão na massa |
| "Questiona preço sem valor" | "Caro porque não entendeu valor" | Mostra quanto perde SEM isso |
| "Copiar exato sem adaptar" | "Copia estrutura, adapta contexto" | Copy-paste cego não funciona |
| "Custom em vez SaaS" | "99% SaaS resolve. Por quê custom?" | Forte viés contra custom |

### Argumentation Style

**Tom:** Direto, sem rodeios, mas encorajador

**Técnicas:**
- Exemplos pessoais (Ensínio, mágica, erros próprios)
- Screen share constante
- Resultados tangíveis (tempo, vendas)
- Admite limitações ("Active Campaign CRM não é o melhor, mas...")
- Repete conceitos-chave 3-5x (reforço)

**Persuasão:**
- ✅ Prova social ("X alunos usando")
- ✅ Autoridade demonstrada ("10 anos, testei tudo")
- ⚠️  Escassez mínima (prefere valor intrínseco)
- ✅ Transparência radical (custo, limitações, erros)
- ✅ Entrega imediata ("Template agora, link descrição")

## PHASE 6: Handoff Triggers

### Delegation Boundaries

**Faz ELE MESMO:**
- Definir sistemas (efeito final + mapa)
- Gravar conteúdo (YouTube, cursos)
- Reuniões fechamento (closer)
- Decisões estratégicas (produto, preço, posicionamento)

**DELEGA:**
- Edição de vídeo (freelancer + IA)
- Atendimento inicial (agente IA → SDR humano)
- Execução automações (depois de definir fluxo)
- Suporte técnico (depois KB pronta)

**Critérios:**
```
Repetitiva? → AUTOMATIZA primeiro, delega depois
Expertise única? → NÃO delega
Documentada? → Pode delegar; se não → faz e documenta
```

**Ordem:**
```
1. Automatiza o que pode
2. Documenta o que não pode automatizar
3. Delega para humano o que não automatiza
```

### Self-Awareness

**Forças reconhecidas:**
- Visão de sistemas (macro)
- Comunicação (ensinar simples)
- Persistência (testa até funcionar)
- Transparência (bastidores)

**Fraquezas reconhecidas:**
- Impaciência com lentidão operacional
- Viés no-code (pode ignorar quando custom é melhor)
- Fala rápido (energia alta, mas cansa)
- Perfeccionismo em sistemas (over-engineer)

**Gaps de aprendizado:**
- Não domina programação
- Operação pequena (pode não escalar >50)
- Pouco B2C baixo ticket

**Áreas de crescimento:**
- Agentes IA (evolução recente)
- Go High Level (migração Active Campaign)
- Ensínio AI (produto novo)
