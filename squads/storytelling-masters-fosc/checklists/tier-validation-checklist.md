# Tier Validation Checklist

Checklist de validação por tier. Use para verificar se o output de um tier específico atende aos padrões de qualidade.

## Tier 0 — Story Structure (McKee + Campbell)

### Structure Integrity
- [ ] Inciting incident presente e claro
- [ ] Gap entre expectativa e resultado é explícito
- [ ] Progressive complications escalam (não são laterais)
- [ ] Crisis apresenta dilema genuíno (não falsa escolha)
- [ ] Climax resolve a questão central com ação
- [ ] Resolution mostra novo equilíbrio (não simplesmente para)
- [ ] Controlling idea expressável em uma frase

### Value Changes
- [ ] Cada beat/cena tem mudança de valor identificável
- [ ] Nenhuma cena é "flat" (sem turning point)
- [ ] Arco geral mostra transformação do protagonista

### Archetypal Depth (se aplicável)
- [ ] Threshold crossing identificável
- [ ] Archetypes são complementares, não conflitantes
- [ ] Retorno com o Elixir está presente

**Pass:** >= 8/12 items ✅

## Tier 1 — Presentation (Duarte + Gallo + Dicks)

### Sparkline Form
- [ ] "What is" e "What could be" claramente definidos
- [ ] Mínimo 3 oscilações entre os dois estados
- [ ] Audiência posicionada como herói (não o apresentador)
- [ ] S.T.A.R. moment presente e posicionado no vale de atenção
- [ ] Call to action específico e acionável
- [ ] "New Bliss" descrito com clareza

### Delivery Excellence
- [ ] Duração respeita limites de atenção (18 min TED ou justificado)
- [ ] Opening hook nos primeiros 18 segundos
- [ ] Rule of Three aplicada em pelo menos um momento
- [ ] Timing por seção documentado

### Personal Story (se aplicável)
- [ ] 5-Second Moment identificado
- [ ] Stakes claros (emocional, relacional, identidade)
- [ ] Começa o mais perto possível do final
- [ ] Usa But/Therefore, não And/Then

**Pass:** >= 9/14 items ✅ (ou >= 6/8 se sem personal story)

## Tier 2 — Persuasion (Cialdini + Heath + Heinrichs)

### Influence Architecture
- [ ] Princípios de influência identificados e posicionados
- [ ] Pre-suasion (priming) planejado antes do ask
- [ ] Ethical gate passed — nenhuma manipulação
- [ ] Sequence lógica: credibilidade → emoção → evidência → ação

### Stickiness
- [ ] Score SUCCESs >= 7/10 em pelo menos 4 dimensões
- [ ] Curse of Knowledge identificado e tratado
- [ ] Mensagem expressável em 10 palavras ou menos (Simple)
- [ ] Pelo menos um pattern break (Unexpected)

### Rhetorical Balance
- [ ] Mix ethos/pathos/logos adequado ao contexto
- [ ] Objeções mapeadas com respostas preparadas
- [ ] Argumento orientado ao futuro (deliberativo, não forense)

**Pass:** >= 8/11 items ✅

## Tier 3 — Specialized

### StoryBrand (Miller — se aplicável)
- [ ] Todos os 7 elementos SB7 preenchidos
- [ ] Customer é o herói (brand é o guide)
- [ ] 3 níveis de problema (external, internal, philosophical)
- [ ] Plan tem 3-4 passos claros
- [ ] One-liner formulado (problem + solution + result)

### Beat Sheet (Snyder — se aplicável)
- [ ] Todos os 15 beats presentes
- [ ] Opening Image ↔ Final Image são espelho
- [ ] Midpoint é false victory ou false defeat
- [ ] All Is Lost é genuinamente o ponto mais baixo
- [ ] Genre classificado com convenções respeitadas

### Public Speaking (Carnegie — se aplicável)
- [ ] Fala a partir de experiência pessoal
- [ ] Preparação documentada além do necessário
- [ ] Notas de confiança e prática incluídas

**Pass:** Varia por framework — ver contagem acima
