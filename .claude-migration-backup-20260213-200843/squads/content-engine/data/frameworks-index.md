## Regras de Uso

1. **Toda decisão no sprint deve citar o framework usado** — sem exceção
2. **Diagnóstico vem ANTES de produção** — Schwartz roda primeiro, sempre
3. **Validação vem DEPOIS de produção** — Torriani e Devil's Advocate são gates finais
4. **Debate usa frameworks como argumentos** — "Segundo o Value Equation do Hormozi..."
5. **Se dois frameworks conflitam**, escalar para o orchestrator com os dois argumentos
