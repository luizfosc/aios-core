# Story 3.2: Template de Relatório

## Description

As a **consultor político consumindo o output da simulação**,
I want **um relatório final em markdown completo e legível por não-técnicos**,
So that **eu possa usar as conclusões diretamente com meu candidato ou equipe de campanha sem precisar interpretar dados brutos**.

## Acceptance Criteria

- [x] AC1: Template `templates/report-template.md` implementado com todas as seções definidas em FR19: Sumário Executivo, Dashboard de Sentimento, Mapa de Argumentos, Análise de Vulnerabilidades, Dinâmica de Opinião, Recomendações, Metodologia
- [x] AC2: Sumário Executivo em 3-5 parágrafos com conclusões acionáveis, tom de consultor de campanha escrevendo para o candidato
- [x] AC3: Dashboard de Sentimento com tabela de distribuição geral (posição, %, intensidade média) E breakdown por segmento demográfico
- [x] AC4: Mapa de Argumentos com argumentos rankeados por impacto, indicando quais segmentos ressoam com cada argumento
- [x] AC5: Análise de Vulnerabilidades com: argumento, segmento mais afetado, taxa de mudança, severidade, recomendação de contranarrativa
- [x] AC6: Seção de Metodologia com disclaimer ético: "Esta é uma simulação exploratória, NÃO uma pesquisa de opinião" + transparência sobre como funciona
- [x] AC7: Metadados incluídos: data, configuração usada, número de agentes, rodadas, contexto ingerido (FR20)
- [x] AC8: Relatório salvo em `output/{run}/report.md` (FR21)
- [x] AC9: Tom do relatório: sem jargão de AI/ML (NFR12). Legível por político ou assessor com nível técnico baixo

## Dev Notes

- **Referência principal:** Arquitetura seção 7.1 (Estrutura do Relatório) — template completo com todas as seções
- **Tom:** Consultor de campanha falando com candidato. Direto, acionável, sem floreios acadêmicos. Pensar como se fosse um relatório de inteligência de campanha
