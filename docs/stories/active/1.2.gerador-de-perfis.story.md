# Story 1.2: Gerador de Perfis de Agentes

## Description

As a **operador da skill politica-app**,
I want **um template que transforme perfis YAML em system prompts cognitivamente ricos**,
So that **cada agente do painel tenha uma persona única com vieses, valores e estilo de expressão distintos**.

## Acceptance Criteria

- [x] AC1: Template `templates/agent-persona.md` implementado conforme seção 4.4 da arquitetura — recebe perfil YAML e gera system prompt com: identidade, classe, escolaridade, valores, prioridades, fontes de informação, estilo de processamento, vieses cognitivos e instruções de resposta em formato YAML estruturado
- [x] AC2: `data/cognitive-biases.yaml` populado com os 8 vieses de Munger relevantes para política (seção 4.2 da arquitetura): reward_punishment, liking_loving, disliking_hating, doubt_avoidance, inconsistency_avoidance, social_proof, authority_influence, contrast_misreaction — cada um com `id`, `name`, `description` e `simulation_effect`
- [x] AC3: `data/influence-principles.yaml` populado com os 6 princípios de Cialdini (seção 4.3 da arquitetura): reciprocidade, compromisso, prova_social, autoridade, escassez, afinidade — cada um com `id`, `name`, `mechanism` e `trigger_condition`
- [x] AC4: `data/demographic-archetypes.yaml` criado com pelo menos 10 arquétipos base cobrindo: classes A/B/C/D/E, regiões (sudeste urbano, nordeste, sul, interior), faixas etárias (18-29, 30-44, 45-59, 60+), religiões (católica, evangélica, sem religião), níveis educacionais
- [x] AC5: Template de persona inclui instruções explícitas para contrariar o viés default do LLM (insight crítico da pesquisa, seção 9 da spec) — agentes conservadores devem usar argumentos genuinamente conservadores, não versões "educadas" do progressismo
- [x] AC6: Template produz output YAML estruturado com campos: `position`, `intensity` (1-5), `main_arguments` (lista), `reasoning`

## Dev Notes

- **Referências diretas:**
  - Arquitetura seção 4.1 (Anatomia de uma Persona) — schema completo do perfil YAML
  - Arquitetura seção 4.4 (Template de System Prompt) — formato exato do template
  - Spec seção 9 (Insights de Pesquisa) — mitigação de viés do LLM é CRÍTICA
  - Spec FR6, FR7, FR8 — requisitos funcionais de geração de agentes
- **Schema do perfil:** `demographic` (age, gender, region, class, education, religion, occupation) + `psychographic` (values, priorities, information_sources, cognitive_biases, influence_susceptibility) + `deliberative` (processing_style, opinion_flexibility, argument_weight, expression_style)
- **Diversidade forçada:** O template DEVE incluir instruções que forcem o LLM a sair do "modo padrão educado progressista". Para classe C/D, usar linguagem coloquial. Para conservadores, usar argumentos de autoridade religiosa/moral genuínos.
- **Restrição:** Output é markdown + YAML. Sem código.
