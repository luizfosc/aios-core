# Story 1.3: Presets Demográficos

## Description

As a **operador da skill politica-app**,
I want **presets demográficos prontos que definam a composição do painel de agentes**,
So that **eu possa rodar simulações com diferentes recortes do eleitorado brasileiro sem configurar cada agente manualmente**.

## Acceptance Criteria

- [x] AC1: Preset `presets/brasil-geral.yaml` criado com 20 agentes distribuídos em 6 segmentos conforme seção 4.5 da arquitetura: classe-ab-urbano-sudeste (3), classe-c-urbano (5), classe-de-periferia (4), rural-interior (3), jovem-digital (3), evangelico-conservador (2)
- [x] AC2: Preset `presets/sp-capital.yaml` criado com 20 agentes focados na região metropolitana de São Paulo, com distribuição proporcional de classes, idades e religiões relevantes para SP
- [x] AC3: Preset `presets/nordeste-evangelico.yaml` criado com 20 agentes focados na região Nordeste com peso maior no segmento evangélico, refletindo crescimento dessa demografia na região
- [x] AC4: Template `presets/custom-template.yaml` criado como guia documentado para criação de presets customizados, com comentários explicativos em cada campo
- [x] AC5: Cada preset contém campos obrigatórios: `name`, `description`, `total_agents`, `composition` (lista de segmentos com `segment`, `count`, `template_overrides`)
- [x] AC6: A soma dos `count` em cada preset é exatamente igual ao `total_agents` declarado
- [x] AC7: Os `template_overrides` de cada segmento são compatíveis com o schema de perfil definido na Story 1.2 (campos de `demographic`, `psychographic`)

## Dev Notes

- **Referência principal:** Arquitetura seção 4.5 (Presets Demográficos) — exemplo completo do `brasil-geral.yaml`
- **Dados demográficos:** Inspirados em IBGE/PNAD/TSE, mas não precisam ser estatisticamente exatos no MVP (decisão documentada na spec)
- **Cada segmento define `template_overrides`** que são aplicados sobre os arquétipos base (`data/demographic-archetypes.yaml`) na hora de gerar os perfis
- **Composição de SP Capital:** considerar distribuição de renda (forte classe C), presença de periferia significativa, diversidade religiosa, jovens digitais
- **Composição Nordeste Evangélico:** considerar crescimento evangélico no NE (~30%), forte presença de classes C/D/E, interior vs. capitais
- **Spec FR3:** Permitir selecionar presets ou composição customizada
- **Spec FR4:** Range de agentes: 10-50 (presets usam 20 como padrão)
