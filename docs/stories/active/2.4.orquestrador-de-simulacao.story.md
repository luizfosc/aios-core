# Story 2.4: Orquestrador de Simulação

## Description

As a **operador da skill politica-app**,
I want **um pipeline completo no SKILL.md que orquestre todo o fluxo de simulação do input ao output**,
So that **eu possa rodar uma simulação completa com um único comando, com progresso visual, checkpoints e tratamento de falhas**.

## Acceptance Criteria

- [x] AC1: SKILL.md contém pipeline completo: Input → Assembly (gerar painel) → Deliberação (N rodadas) → saída com dados brutos
- [x] AC2: Pipeline aceita como input: tema/pergunta (obrigatório), preset demográfico (default: brasil-geral), contexto opcional (markdown/texto), número de agentes e rodadas (configuráveis via config.yaml)
- [x] AC3: Progresso visual durante execução: "Fase 2/4: Deliberação — Rodada 2/3 — Batch 1/2 — Agente 5/10" (NFR3)
- [x] AC4: Checkpoints entre rodadas: cada fase grava output antes de avançar. Se falhar, é possível retomar do último checkpoint (NFR9)
- [x] AC5: Tratamento de falhas implementado: output malformado → marcar error, timeout → marcar timeout, >30% falhas → abortar com mensagem clara
- [x] AC6: Config snapshot salvo em `output/{run}/config-snapshot.yaml` no início da simulação (FR22)
- [x] AC7: Pasta de output segue padrão timestamped: `output/{YYYYMMDD-HHMM}-{slug}/` com `panel.yaml`, `rounds/`, `config-snapshot.yaml`
- [x] AC8: Mensagens de erro claras e acionáveis em português (NFR11)

## Dev Notes

- **Referências diretas:**
  - Spec seção 3.4 (Fluxo de Dados End-to-End) — diagrama completo do pipeline
  - Spec FR1-FR5 (Configuração de Simulação)
  - Spec NFR1 (max 15 min wall-clock para 20 agentes, 3 rodadas)
  - Spec NFR7-NFR9 (Confiabilidade: idempotência, falhas, checkpoints)
  - Arquitetura seção 3.4 (Tratamento de Falhas) — tabela de cenários
- **O SKILL.md É o "código":** Toda a lógica de orquestração é escrita como instruções markdown que o Claude Code segue. Não há script executável
- **Fases do pipeline:**
  1. Validar inputs (tema obrigatório, preset existe, config válida)
  2. Assembly: gerar painel (ou carregar painel existente se fornecido)
  3. Loop de rodadas: para cada rodada R de 1 a N → deliberar → sumarizar → checkpoint
  4. Salvar dados brutos
- **Checkpoints:** Após cada rodada, o `round-N.yaml` é salvo. Se o processo falhar, pode-se retomar lendo os rounds existentes e continuando do próximo
- **Nota:** Esta story NÃO inclui agregação e relatório (Epic 3). O pipeline para aqui nos dados brutos
