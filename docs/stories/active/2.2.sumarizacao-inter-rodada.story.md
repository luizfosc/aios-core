# Story 2.2: Sumarização Inter-Rodada

## Description

As a **operador da skill politica-app**,
I want **que os argumentos de cada rodada sejam sumarizados automaticamente entre rodadas**,
So that **os agentes nas rodadas seguintes recebam um resumo equilibrado dos argumentos apresentados, simulando exposição a opiniões diversas**.

## Acceptance Criteria

- [x] AC1: Template `templates/summarizer.md` implementado com instruções para condensar argumentos de uma rodada
- [x] AC2: Sumarização agrupa argumentos por posição (a_favor, contra, neutro) — sem misturar posições
- [x] AC3: Argumentos rankeados por frequência dentro de cada posição (quantos agentes usaram argumento similar)
- [x] AC4: Máximo de 5 argumentos por posição no sumário (os mais frequentes) — spec seção 5.3
- [x] AC5: Sumário inclui distribuição percentual: "A favor: X% | Contra: Y% | Neutro: Z% | Indeciso: W%"
- [x] AC6: Linguagem do sumário é acessível (não acadêmica) — compatível com agentes de baixa escolaridade
- [x] AC7: Sumarização executada pelo agente principal (Claude Code), NÃO por sub-agente — decisão de arquitetura seção 5.3

## Dev Notes

- **Referência principal:** Arquitetura seção 5.3 (Sumarização Inter-Rodada) — regras exatas de sumarização
- **Por que pelo agente principal?** Sumarização é tarefa sequencial simples. Spawnar sub-agente adiciona latência sem benefício (decisão documentada na arquitetura)
- **Input:** `round-N.yaml` da rodada que acabou de completar
- **Output:** Texto markdown formatado para injeção no template de deliberação da próxima rodada
- **Cuidado:** O sumário deve ser NEUTRO — não deve favorecer nenhum lado. Deve apresentar argumentos de todas as posições com igual destaque
- **Cuidado:** Usar linguagem acessível. Agentes de classe C/D com escolaridade fundamental precisam entender o sumário
- **Spec FR12:** "Nas rodadas 2+, cada agente recebe sumário dos argumentos das rodadas anteriores"
