# Story 2.1: Rodada 1 — Opinião Individual

## Description

As a **operador da skill politica-app**,
I want **que cada agente do painel forme sua opinião individual sobre o tema sem exposição a outros agentes**,
So that **a primeira rodada capture a reação "pura" de cada perfil demográfico antes de qualquer influência social**.

## Acceptance Criteria

- [x] AC1: Template `templates/deliberation-round-1.md` implementado com instruções para o agente formar opinião individual — recebe perfil + tema + contexto opcional e instrui resposta em YAML estruturado
- [x] AC2: Output YAML de cada agente contém campos obrigatórios: `position` (a_favor|contra|neutro|indeciso), `intensity` (1-5), `main_arguments` (lista de 2-3 strings), `reasoning` (string)
- [x] AC3: Spawn de agentes via Agent tool em paralelo, com batches de 10 (conforme config.yaml `batch_size`)
- [x] AC4: Outputs de todos os agentes salvos em `output/{run}/rounds/round-1.yaml` com estrutura: lista de objetos com `agent_id`, `agent_name`, `segment`, `response` (o YAML de resposta), `status` (success|error|timeout)
- [x] AC5: Agentes com output malformado são marcados como `status: error` e não bloqueiam a rodada (spec seção 3.6, tratamento de falhas)
- [x] AC6: Se mais de 30% dos agentes falharem, abortar rodada e alertar usuário com mensagem clara
- [x] AC7: Progresso visual mostrado durante execução: "Rodada 1 — Batch 1/2 — Agente 5/10" (NFR3)

## Dev Notes

- **Referências diretas:**
  - Arquitetura seção 5.1 (Ciclo Deliberativo — Rodada 1: Formação Individual)
  - Arquitetura seção 3.2 (Ciclo de Vida de um Agente — SPAWN → PROMPT → RESPOND → CAPTURE → TERMINATE)
  - Arquitetura seção 3.3 (Estratégia de Paralelismo)
  - Spec FR10, FR11, FR13
- **Mecanismo de spawn:** Cada agente é um sub-agente via Agent tool. O system prompt é gerado pelo template `agent-persona.md` (Story 1.2). O user prompt é o tema + contexto, formatado pelo template `deliberation-round-1.md`
- **Batching:** Para painéis > 10 agentes, dividir em batches de `batch_size` (default: 10). Batches executam em paralelo internamente, mas são sequenciais entre si
- **Stateless:** Agentes morrem após responder. Não há sessão persistente (decisão de arquitetura)
- **Parsing:** O output do Agent tool deve ser parseado para extrair o YAML de resposta. Se parsing falhar, marcar como error
- **Restrição CON-7:** Máximo 50 agentes por simulação
