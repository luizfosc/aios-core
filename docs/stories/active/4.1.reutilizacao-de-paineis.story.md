# Story 4.1: Reutilização de Painéis

## Description

As a **operador da skill politica-app**,
I want **poder carregar um painel de agentes de uma simulação anterior e reutilizá-lo com um tema diferente**,
So that **eu possa comparar como o mesmo grupo de "pessoas virtuais" reage a temas diferentes, mantendo controle das variáveis**.

## Acceptance Criteria

- [x] AC1: SKILL.md aceita parâmetro opcional `panel_path` que aponta para um `panel.yaml` existente
- [x] AC2: Quando `panel_path` é fornecido, a fase de Assembly é pulada — perfis são carregados diretamente do arquivo
- [x] AC3: Validação de integridade: verificar que o `panel.yaml` carregado contém todos os campos obrigatórios do schema (id, name, demographic, psychographic, deliberative)
- [x] AC4: Se validação de integridade falhar, mensagem de erro clara indicando quais campos estão faltando
- [x] AC5: O `config-snapshot.yaml` da nova simulação registra que o painel foi reutilizado (campo `panel_source: {path}` em vez de `panel_source: generated`)
- [x] AC6: O relatório final menciona que o painel foi reutilizado: "Painel reutilizado de simulação anterior: {path}"

## Dev Notes

- **Referência:** Spec FR23 — "Permitir reutilizar painéis de agentes entre simulações"
- **Spec FR9:** "Armazenar perfis gerados em panel.yaml para auditoria e reuso" — esta story implementa o "reuso"
- **Mecânica simples:** O SKILL.md já sabe gerar painel na fase de Assembly. Aqui, basta adicionar um branch condicional: se `panel_path` fornecido → ler arquivo, senão → gerar
- **Validação:** O schema do panel.yaml deve ser o mesmo independente de ser gerado ou carregado. Validar campos obrigatórios antes de prosseguir
- **Caso de uso principal:** Rodar "reforma tributária" com o mesmo painel de "maioridade penal" para ver como o mesmo grupo reage a temas diferentes
