# Story 4.2: Cenários Comparativos

## Description

As a **consultor político**,
I want **rodar múltiplas simulações com o mesmo painel e temas diferentes, e obter um relatório comparativo**,
So that **eu possa visualizar como a opinião do mesmo grupo muda conforme o tema ou enquadramento da questão**.

## Acceptance Criteria

- [x] AC1: SKILL.md aceita modo "comparativo" que recebe: painel fixo (panel_path) + lista de 2-3 temas/perguntas
- [x] AC2: Cada tema é executado como simulação independente, reutilizando o mesmo painel
- [x] AC3: Ao final, relatório comparativo gerado mostrando deltas entre cenários: diferença de distribuição de sentimento por tema, argumentos que aparecem em um cenário mas não no outro, segmentos que mudam de posição entre cenários
- [x] AC4: Relatório comparativo salvo em `output/{timestamp}-comparativo/comparison-report.md`
- [x] AC5: Dados individuais de cada cenário preservados nas subpastas respectivas para auditoria

## Dev Notes

- **Referência:** Spec FR24 — "Permitir cenários comparativos (mesmo painel, perguntas diferentes)"
- **Spec Story 4.2:** "Rodar múltiplas simulações com mesmo painel + perguntas diferentes. Gerar relatório comparativo mostrando deltas entre cenários"
- **Mecânica:** Basicamente são N simulações com `panel_path` fixo (Story 4.1) + um template adicional para o relatório comparativo
- **Relatório comparativo deve responder:**
  - Em qual tema o painel é mais dividido?
  - Quais segmentos mudam de posição entre temas?
  - Quais argumentos são transversais (aparecem em todos os cenários)?
  - Quais argumentos são específicos de um tema?
- **Caso de uso:** Testar 3 enquadramentos diferentes de uma mesma proposta para ver qual gera menor resistência
