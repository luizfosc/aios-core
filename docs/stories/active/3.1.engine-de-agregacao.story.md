# Story 3.1: Engine de Agregação

## Description

As a **operador da skill politica-app**,
I want **que os dados brutos das rodadas de deliberação sejam agregados em métricas quantitativas**,
So that **eu tenha uma visão consolidada de distribuição de sentimento, ranking de argumentos, vulnerabilidades e clusters de opinião**.

## Acceptance Criteria

- [x] AC1: Template `templates/aggregator.md` implementado — lê todos `round-*.yaml` e gera `aggregation.yaml`
- [x] AC2: Distribuição de sentimento calculada: % a_favor/contra/neutro/indeciso, tanto geral quanto por segmento demográfico
- [x] AC3: Intensidade média calculada por posição (identifica se "contra" é mais veemente que "a favor")
- [x] AC4: Ranking de argumentos por frequência E por `impact_score` — onde `impact_score = count(agentes que citaram o argumento como change_trigger)` (FR16)
- [x] AC5: Vulnerabilidades identificadas: argumentos com `impact_score >= ceil(0.2 x segment_size)` são marcados como pontos fracos narrativos (FR17) — cada vulnerabilidade com: argumento, segmento mais afetado, taxa de mudança, severidade, recomendação de contranarrativa
- [x] AC6: Clusters de opinião identificados via análise semântica: agentes agrupados por similaridade de argumentos, com mínimo de 3 agentes por cluster, rotulados por tema predominante e demográficos
- [x] AC7: Dinâmica de opinião documentada: mudanças entre rodadas com gatilho identificado (round_1_to_2, round_2_to_3, etc.)
- [x] AC8: `aggregation.yaml` salvo em `output/{run}/aggregation.yaml` com estrutura conforme exemplo da seção 6.1 da arquitetura

## Dev Notes

- **Referências diretas:**
  - Arquitetura seção 6.1 (Métricas Calculadas) — exemplo completo do `aggregation.yaml`
  - Arquitetura seção 6.2 (Cálculo de Impact Score) — fórmula exata
  - Arquitetura seção 6.3 (Detecção de Clusters) — processo de 4 passos
  - Spec FR15-FR18 (Requisitos de Agregação)
  - Spec seção 3.9 (Motor de Agregação)
- **Impact Score:** `impact_score = count(agents where change_trigger contains argument)`. Argumentos com score alto são vulnerabilidades
- **Threshold de vulnerabilidade:** `ceil(0.2 x segment_size)`. Para segmento de 5 agentes → impacto >= 1. Para segmento de 10 → impacto >= 2
- **Clusters:** Feitos por prompt (análise semântica via LLM), NÃO por algoritmo. O LLM agrupa agentes que usaram argumentos tematicamente similares
- **Cuidado:** A agregação é feita pelo agente principal, não por sub-agente. É uma tarefa de análise dos dados já coletados
- **Spec NFR4:** "Pelo menos 10 argumentos únicos" — a agregação deve preservar diversidade
