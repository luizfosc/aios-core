# Story 3.5: Simulação Completa de Validação

## Description

As a **operador da skill politica-app**,
I want **rodar uma simulação realista completa de ponta a ponta com 20 agentes, 3 rodadas e contexto ingerido**,
So that **eu valide que o pipeline inteiro funciona e que o output é útil para um consultor político real**.

## Acceptance Criteria

- [x] AC1: Simulação executada com cenário canônico: tema "Proposta de redução da maioridade penal para 16 anos" + contexto mínimo de 2 notícias recentes (max 300 palavras cada)
- [x] AC2: Painel de 20 agentes usando preset `brasil-geral`, 3 rodadas de deliberação
- [x] AC3: `report.md` gerado com TODAS as seções de FR19: Sumário Executivo, Dashboard de Sentimento, Mapa de Argumentos, Vulnerabilidades, Recomendações, Metodologia
- [x] AC4: `aggregation.yaml` contém pelo menos 10 argumentos únicos (NFR4)
- [x] AC5: Pelo menos 2 segmentos demográficos com distribuição de sentimento distinta — diferença >= 20 pontos percentuais entre segmentos em alguma posição
- [x] AC6: Nenhum agente tem posição final idêntica ao agente imediatamente anterior na ordem de processamento
- [x] AC7: Wall-clock total <= 15 minutos (NFR1)
- [x] AC8: Review qualitativo: recomendações são acionáveis e argumentos são plausíveis dentro do perfil demográfico

## Dev Notes

- **Referência:** Spec Story 3.5 — cenário canônico com ACs mensuráveis (FIX-QA-CRIT-2, FIX-QA-CRIT-3)
- **Tema canônico fixo:** "Proposta de redução da maioridade penal para 16 anos". Este tema serve como benchmark qualitativo comparável entre iterações da skill
- **Contexto mínimo:** 2 notícias recentes sobre o tema (preparar como arquivos markdown em `data/` ou `output/`)
- **Validação quantitativa (ACs 3-6):** Estas são métricas objetivas. Passar ou não passar
- **Validação qualitativa (AC8):** Subjetiva, mas necessária. Perguntas-chave:
  - As recomendações são algo que um consultor real usaria?
  - Os argumentos fazem sentido para o perfil demográfico de cada agente?
  - O relatório é legível por um não-técnico?
- **Esta é a validação final do MVP.** Se esta simulação funciona e gera output útil, o politica-app está pronto para uso
- **Spec NFR1:** 15 minutos max para 20 agentes, 3 rodadas
- **Output:** Salvar em `output/{timestamp}-validacao-completa/`
