# Story 3.3: Gerador de Recomendações

## Description

As a **consultor político consumindo o relatório**,
I want **recomendações estratégicas contextualizadas derivadas dos dados da simulação**,
So that **eu tenha ações concretas para sugerir ao candidato, não apenas dados descritivos**.

## Acceptance Criteria

- [x] AC1: Template `templates/recommendations.md` implementado — transforma dados agregados em 3-5 recomendações estratégicas
- [x] AC2: Cada recomendação contém: ação sugerida, justificativa (baseada nos dados), segmento-alvo principal, nível de urgência (crítica/alta/média/baixa)
- [x] AC3: Recomendações são contextualizadas ao tema da simulação (não genéricas tipo "melhore a comunicação")
- [x] AC4: Pelo menos 1 recomendação aborda diretamente a vulnerabilidade de maior impacto identificada na agregação
- [x] AC5: Recomendações são acionáveis: descrevem O QUE fazer, PARA QUEM e POR QUÊ (não apenas "considere X")
- [x] AC6: Tom: consultor de campanha dando conselho prático, sem jargão de AI

## Dev Notes

- **Referência:** Spec Story 3.3 — "Template recommendations.md que transforma dados agregados em 3-5 recomendações estratégicas contextualizadas"
- **Input:** `aggregation.yaml` — especialmente as seções de vulnerabilidades, clusters e dinâmica de opinião
- **Fórmula implícita:** Recomendações devem ser o "so what?" dos dados. Se a vulnerabilidade #1 é "vai aumentar imposto pra classe média", a recomendação deve ser algo como "Priorizar contranarrativa com dados concretos de isenção para classe C, direcionada ao segmento classe-c-urbano, urgência crítica"
- **Spec risco:** "Output não acionável para consultores" (Probabilidade: Média, Impacto: Crítico). Este template é a principal mitigação desse risco
- **Diferenciação:** Recomendações são o que separam este produto de uma simples pesquisa de sentimento. Devem demonstrar valor estratégico
