# Story 3.4: Ingestão de Contexto

## Description

As a **operador da skill politica-app**,
I want **poder fornecer documentos de contexto (notícias, pesquisas, posts) para enriquecer a simulação**,
So that **os agentes formem opiniões baseadas em informações reais e atuais, não apenas no conhecimento geral do LLM**.

## Acceptance Criteria

- [x] AC1: Pipeline de ingestão aceita documentos de contexto opcionais nos formatos: markdown, texto puro, YAML (FR2)
- [x] AC2: Cada documento de contexto é sumarizado para no máximo 500 palavras antes de ser injetado nos prompts dos agentes — restrição de arquitetura para preservar context window
- [x] AC3: Múltiplos documentos de contexto podem ser fornecidos (lista de paths) e são concatenados após sumarização
- [x] AC4: O contexto sumarizado é injetado nos prompts de TODAS as rodadas de deliberação (rodada 1 e rodadas 2+)
- [x] AC5: SKILL.md atualizado com instruções de como fornecer contexto: path para arquivo(s) ou texto inline
- [x] AC6: Se nenhum contexto fornecido, simulação roda normalmente sem contexto adicional (comportamento graceful)
- [x] AC7: Contexto ingerido é documentado nos metadados do relatório (FR20): lista de documentos, tamanho original, tamanho sumarizado

## Dev Notes

- **Referência:** Spec Story 3.4, Spec FR2
- **Por que sumarizar?** Documentos longos consomem context window e limitam o número de agentes paralelizáveis por batch. A sumarização para 500 palavras é restrição de arquitetura documentada na spec (FIX-QA-REC-2)
- **Ordem no pipeline:** A ingestão de contexto acontece ANTES da Assembly. O contexto sumarizado é passado como input para os templates de deliberação
- **Pode começar em paralelo:** Esta story depende apenas do scaffold (1.1), não do motor de deliberação. A spec sugere que pode ser desenvolvida em paralelo com o Epic 2
- **Spec OQ-6:** "Limite de conhecimento temporal do LLM impacta cenários de política atual?" — A ingestão de contexto é a mitigação principal
- **WebSearch/WebFetch:** São opcionais para ingestão de contexto adicional (spec seção 8), mas no MVP a ingestão é por arquivo local
