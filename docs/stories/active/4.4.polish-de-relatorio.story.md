# Story 4.4: Polish de Relatório

## Description

As a **consultor político consumindo o relatório final**,
I want **um relatório com polish profissional, disclaimer ético robusto e formatação otimizada**,
So that **eu possa apresentar o output diretamente ao candidato ou à equipe de campanha com confiança**.

## Acceptance Criteria

- [x] AC1: Disclaimer ético expandido e robusto: explica claramente que é simulação, não pesquisa; limitações do método; não deve ser única base para decisões; viés inerente do LLM reconhecido
- [x] AC2: Seção de Metodologia expandida: explica em linguagem acessível como a simulação funciona (painel virtual → deliberação → agregação), sem jargão técnico de AI
- [x] AC3: Formatação otimizada para conversão em PDF: headings claros, tabelas bem formatadas, espaçamento consistente, sem elementos que quebrem em PDF
- [x] AC4: Sumário Executivo refinado: mais conciso, com bullet points das conclusões-chave nos primeiros parágrafos
- [x] AC5: Template de relatório inclui footer com: "Gerado por politica-app | Simulação exploratória — não substitui pesquisa de opinião profissional"
- [x] AC6: Refinamentos baseados em feedback da simulação de validação (Story 3.5): se algo no relatório não ficou claro ou acionável, ajustar aqui

## Dev Notes

- **Referência:** Spec Story 4.4 — "Refinamento do template com base em feedback. Disclaimer ético mais robusto. Seção de metodologia expandida. Formatação otimizada para impressão/PDF"
- **Spec risco:** "Credibilidade excessiva do usuário" (Probabilidade: Média, Impacto: Alto). O disclaimer é a principal mitigação
- **Spec risco:** "Questões éticas sobre simulação de opinião" (Probabilidade: Média, Impacto: Alto). Transparência na metodologia é a mitigação

- **Esta é a última story do MVP.** Após o polish, a skill está pronta para uso
