# Story 2.5: Dry-Run End-to-End

## Description

As a **operador da skill politica-app**,
I want **rodar uma simulação completa de deliberação com painel reduzido para validar o pipeline inteiro**,
So that **eu tenha confiança de que assembly + deliberação + checkpoints funcionam corretamente antes de avançar para agregação**.

## Acceptance Criteria

- [x] AC1: Simulação executada com 10 agentes e 2 rodadas usando preset `brasil-geral` (metade do painel padrão)
- [x] AC2: Assembly gera `panel.yaml` com 10 perfis completos e válidos
- [x] AC3: Rodada 1 gera `rounds/round-1.yaml` com respostas estruturadas de todos os agentes (ou com status error/timeout documentado)
- [x] AC4: Sumarização entre rodadas gera sumário com argumentos agrupados por posição e distribuição percentual
- [x] AC5: Rodada 2 gera `rounds/round-2.yaml` com campos adicionais de mudança de opinião (`opinion_changed`, `change_trigger`, `previous_position`)
- [x] AC6: Checkpoints funcionam: `panel.yaml`, `round-1.yaml` e `round-2.yaml` existem como arquivos independentes no disco
- [x] AC7: Config snapshot salvo em `config-snapshot.yaml` no início da simulação
- [x] AC8: Wall-clock total documentado (meta: < 10 minutos para 10 agentes, 2 rodadas)

## Dev Notes

- **Referência:** Spec Story 2.5 — "Simulação completa com 10 agentes e 2 rodadas"
- **Objetivo:** Validação funcional do pipeline, não qualidade dos argumentos (isso foi validado no Spike)
- **O que verificar:**
  - Assembly funciona? `panel.yaml` gerado corretamente?
  - Deliberação gera dados estruturados? YAML parseável?
  - Checkpoints salvam corretamente? Arquivos existem no disco?
  - Progresso visual funciona? Mensagens aparecem durante execução?
  - Tratamento de falhas: se algum agente falhar, os outros continuam?
- **Spec NFR1:** Meta de performance: 15 min para 20 agentes, 3 rodadas. Para 10 agentes, 2 rodadas, esperar < 10 min
- **Output:** Salvar em `output/{timestamp}-dry-run-e2e/`
